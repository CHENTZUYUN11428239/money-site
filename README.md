# money-site
網站連結 https://chentzuyun11428239.github.io/money-site/



要寫一個 **可以共用的記帳網站**（多人／團體／公司皆可用），基本上你可以把整個計畫分成幾個重要部分：**需求規劃 → 系統設計 → 實作 → 上線 → 運營與維護**。以下用清楚的架構整理給你：

---

## 🧠 一、先釐清需求（最重要！）

### ✔ 功能定位

你要先問自己／團隊：

📌 這網站要解決什麼痛點？
📌 主要使用者是誰？（公司／社團／家庭／合作團隊）
📌 重點功能有哪些？（例如：共同帳目、報表分析、預算、權限管理）

---

## 💡 二、核心功能清單（必備）

### 🧾 1. 使用者系統

* 註冊／登入
* 權限與角色管理（例如：管理員、出納、一般成員）
* 團隊邀請與成員管理

### 💰 2. 帳目管理

* 建立／編輯／刪除花費
* 多人共用分類、標籤、項目
* 帳本分層（公司 > 部門 > 小組）

### 📊 3. 報表與統計

* 月報／年度報表
* 團隊總花費 vs 個人貢獻
* 圖表顯示（長條圖、圓餅圖）

### 📅 4. 預算與提醒

* 設定預算
* 超支提醒

### ☁️ 5. 資料同步與安全

* 多裝置同步
* 權限分級（誰可以看／誰可以改）
* 自動備份與匯出（CSV、Excel、PDF）

### 🔐 6. 系統管理

* 系統設定
* 日誌與審計（誰改過什麼）

## 📌 五、類似市面上的產品

是的，**市面上有很多類似的記帳工具**，但定位不同：

### ✔ 個人／家庭類

* **Money Lover**
* **You Need A Budget (YNAB)**
* **Goodbudget**
* **記帳城市 / 帳務易**

### ✔ 多人／團隊協作記帳（比較稀少）

這比較接近企業或團體記帳需求，但很多是付費服務：

* **Splitwise**（多人分帳，但不完整是帳務系統）
* **Zoho Books / QuickBooks / Xero**（企業會計，但比較專業）
* **Wave Accounting**

這中間其實有一個產品機會：
👉 介於「多人分帳」和「公司記帳（會計）」之間，**既簡單又能多人共用、可見報表與審計**的系統。

## 🎯 六、要注意的商業面

* 免費 + 進階付費功能
* 企業月租制

### ✅ 資安跟法規

* 資料加密（SSL / HTTPS）
* GDPR／個資保護法
* 備援機制與資料留存政策

## 📌 小總結

建立一個多人共用的記帳網站，**不只是記帳**，還要能處理：

✔ 使用者身份與權限
✔ 多人共用帳本
✔ 視覺化報表
✔ 安全 + 同步

市面上有許多產品，但很少是 **真正針對「團體或公司」的共同記帳解決方案**。如果能做到簡單、清楚、可共用，那就是有競爭力的產品。
